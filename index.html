Getting Started

Great, you’ve downloaded Rustcord, but you’ll still need to have the following

steps completed before setting up the Rustcord plugin:

  1. Download and install the Discord Rust Extension on your Rust server.
  2. Setup a bot for Rustcord to use on your Discord server.
  3. Enable Developer Mode in your Discord client.

After that’s done, proceed on.

Plugin Compatibility

This is a list of other compatible plugins. Any Discord related plugins on this list have been verified to run alongside Rustcord without conflict. Any Discord related plugins not on this list may break certain features of this plugin, or the entire plugin.

        Compatible Plugins:

  • DiscordAuth
  • DiscordRewards
  • DiscordSync

User Permissions

The following plugin specific user permissions are available:

  • rustcord.hidejoinquit         - Player’s joins & quits won’t be logged to Discord.
  • rustcord.hidechat                  - Player’s chat won’t be logged to Discord.

Configuration (General Settings)

First we’ll start with the general settings in the config file.

Apikey - This is where you put the token for the Discord bot you made in the Getting Started section.

AutoReloadPlugin - Whether or not you want the plugin to reload itself periodically. The Discord extension likes to drop connections often, so we recommend this option until that is addressed. If you leave this false, we recommend periodic reloads by a third party plugin, such as TimedExecute.

AutoReloadTime - Time (in seconds) between auto reloads if enabled above.

EnableBotStatus - Enables custom bot status on Discord (customizable in lang file).

GameChatTag - Allows you to customize the [Rustcord] tag in Discord>Game chat.

GameChatTagColor - Color of [Rustcord] tag, hex style (#123456).

GameChatNameColor - Name color for person Discord>Game chatting, hex style (#123456).

GameChatTextColor - Color for message from Discord>Game, hex style (#123456).

CommandPrefix - The prefix you want to use for Discord channel commands. The default is !, so the players command on discord would be !players.

ReportCommand - Allows you to change /report to /whateveryouwant, in case the command interferes with another plugin you run also using /report.

FIlteredWord - Replacement word for any words on your filter list.

FilteredWordList - A list of words you want filtered out of your chat relay.

LogExcludeGroups - List any groups here you don’t want logged when people are added to or removed from. ALWAYS keep ‘default’ in your group list with any others you add or remove.

LogExcludePerms - List any permissions you don’t wanted logged when they’re given or revoked. This is useful if you have zones that constantly give and take permissions from players.

EnableCustomLogging - When enabled, this will relay ANY console line that contains the keyword(s) listed in your CustomLoggingKeywords list to a single channel with log_console flag.

CustomLoggingKeywords - Add keywords of lines you want logged to your log_console channel here. For example, if you put “[Advert Messages]” in the list, any console lines with “[Advert Messages]” in them will be relayed to Discord. Format is the same as the other lists.

WARNING: Use caution when using the custom logging feature. Enabling it for items that occur too often will trigger Discord rate limiting.

Configuration (Log Settings)

These settings tell the plugin what hooks to watch, NOT what to relay, that’s what the channel flags (see channel flags section) do. If LogChat is true, but you don’t have msg_chat assigned, the plugin will watch player chat but not send it anywhere. If you have msg_chat assigned to a channel, but LogChat set to false, the plugin won’t relay chat, as it’s not watching chat hooks.

Configuration (Channels)

The channel area of the config is where you set up the actual relays from game to Discord, or vice versa. You can customize you output to as many or as few channels as you’d like.

Channelid - To get your channel ID, right click the channel in your channel list on Discord while in Developer Mode, and click “copy id”.

Perms - This is where you list the specific channel flags (see table below) that you want for the channel. You are given some basic flags in the generated default config.

Configuration (Command Roles)

The command roles section dictates who is allowed to use specified commands. The roles you put in this config must match the role names on your Discord. You may also leave the role list empty if there is a command you want to allow everyone to use. This section is important, as it will limit access to commands you don’t want everyone to have on Discord. Commands you don’t list in here will act as if they have an empty list (as though everyone can use them).

"Commandroles":{

        "players":[ ],

        "kick":[

                Admins,

                Staff,

                Mods

]

In the above example, the list for players is left empty, so anyone with access to type in a channel with the cmd_players flag will be able to use !players.

You can also see that the list for kick is populated with Admins, Staff & Mods, so only Discord users that have one of those roles assigned to them on your Discord server would be able to use the !kick command in a channel with the cmd_kick flag.

Language File

The language file is where you can customize your Discord output lines for every action Rustcord relays to your Discord channels (except DeathNotes, see FAQ). It supports all built in Discord emojis by default, and can also support custom emojis. Fresh language files spawn with every field option loaded into them (such as {players}).

Custom Emojis

The language file supports use of custom emojis, though the method is a bit more tedious.
I will use the Rustcord Discord for this example, where :chopper: =

  1. Upload your custom emoji to your Discord server (Server > Server Settings > Emoji).
  2. Find the full ID of your emoji using this command in Discord: \:your_emoji:
  3. The information it returns is what you use in your lang file.

So for example, \:chopper: in the Rustcord Discord returns <:chopper:579227516686827520>. That entire line is what you need for your lang line. Below is a proper lang line using this example information:

"RUST_OnHeliSpawn": "<:chopper:579227516686827520> Patrol Helicopter has spawned."

This method may be used for any of the language file lines.

Commands

Game Commands

These commands are used by players in game to relay to Discord. Game commands require the game_* channel flags to relay to Discord.

Report: Allows players to make reports to staff. Customizable in config.

Format: /report <message>

Bug: Functions like report, not configurable.

Format: /bug <message>

Discord Commands

These commands are used in Discord to interact with the server. Discord commands require the cmd_allow AND cmd_<command you want to use> channel flags to be used in specified channel(s).

Players: Shows current player count, with player list.

Format: !players

Kick: Kick a player from the game server.

Format: !kick <name/id> <reason>

Ban/Unban/Timeban: Ban, Unban, or Time Ban a player.

Formats:

        !ban <name/id> <reason>

        !unban <name/id>

        !timeban  <steamid> <name> <time (1s/1m/1h/1d)> <reason>

Note: !timeban requires BOTH SteamID AND name fields, also do NOT add quotes around reasons with multiple words.

Correct: this is the ban reason

Incorrect: “this is the ban reason”

Com: Used for sending any raw string of text directly to server console.

Format: !com <command line>

Mute/Unmute: Used to mute/unmute players in game from Discord.

Formats:

        !mute <name/id> <time (optional) (1s/1m/1h/1d)> <reason (optional)>

        !umute <name/id>

Note: Requires BetterChatMute

F.A.Q.

Q: How do I get a channel ID for the config?

A: Enable Developer Mode in Discord (User Settings > Appearance > Advanced). Then in Discord, right click your channel in the left hand side channel list and chose Copy ID (last option in drop down).

Q: Death messages aren't in the lang file, how do I customize them?

A: The death messages are taken from your DeathNotes plugin config, and will mirror whatever is set there.

Q: I use KillFeed instead of DeathNotes, how can I get deaths to show?

A: Unfortunately right now, Rustcord REQUIRES DeathNotes to relay deaths to Discord. However, if you're using KillFeed, DeathNotes WILL run along side it and shouldn't cause issues. Just set "Show Kills in Chat" to false in the DeathNotes config, and you'll get deaths on Discord without deaths from DeathNotes in your chat feed. It's not the most ideal setup, but it works.

Q: The plugin failed to compile on server start/restart.

A: Check to make sure you're not running other Discord plugins. Rustcord will still work with them installed, but will need to be reloaded after server load (TimedExecute could do this). If you go that route, the initialized message wont work on restarts and wipes, as the plugin isn't loaded on startup.

Q: How does 2 way chat work?

A: Anything typed by anyone in a channel with the msg_chat flag will automatically be sent into the game chat with a [Discord] chat tag. It's automatic and doesn't require a command. We recommend servers with higher populations to restrict it to staff only able to type, but everyone able to view.
